1.JavaScript(build by v8 engine)(js parses->convert into bytecode->JIT->execute)
It is a interpreted language scripting language used to create webapplications interactive and dynamic. JS interpreted it dosen't need external compilation, js uses Just-In-Time compilation for execution.

2.JIT (Just-In-Time)
Technique where the js code is converted to machine code at runtime instead of before execution.js is converted into AST -> AST converted into bytecode->JIT convert in to machine code-> execution.

3.AST
Tree structure that represents syntax of our code in a structured and hierrarchial way.
 JavaScript Execution Flow 🚀

1. AST (Abstract Syntax Tree)
   - Represents JavaScript code in a structured, hierarchical tree format

2. Bytecode Generation
   - JavaScript engine converts the AST into bytecode

3. Interpreter
   - Executes the bytecode immediately (fast startup)

4. JIT (Just-In-Time Compiler)
   - Detects frequently executed (“hot”) code
   - Compiles it into optimized machine code
   - Improves performance significantly ⚡

-> Inline Caching - That remember the value and directly jumps without lookup.

Function - It is a block of code that can be reused everywhere intead of rewriting.

6.Scopes.
Scope - it determines the visibility of data. Determines where a function or variables can be accessed.
a.Functional Scope - Variables can be only accessed inside the function.(var).
b.Block Scope - Variables can be only accessed inside blocks{}.(let,const).
c.Global Scope - Variables can be accessed every where in the file.
d.Lexical Scope(scope where the function is defined) - A function that can access the variables that defined in the place where the function was written, not where it is called.
e.Local Scope - Scope inside function.(variables are decleared inside the function are only accissible within the function).

 Event loop is a mechanism that continuiosly check the call stack and handles asynchronous tasks.
 WEB APIs-> JS alone cannot handle timer functions, fetchs, events it is provied by runtime environments(Brower apis), to when it encounter it is passed to WEB APIs to handle it asynchronously else js will block the main thread. After hanling it is passed to the callback queue.

 MicroTask queue -> contain the Promise and it has higher priority. Waits for completion of synchronous tasks then pushed to call stack immediatly before call back queue.

23.Currying
 Functions with multiple arugments are passed to a sequence of functions to improve controll over data and reusability and partial application.
 Currying for infinite sum:
 
 function sum(a){
    return function (b){
        if(b===undefined)return a;
        return sum(a+b);
    }
}
console.log(sum(10)(20)())
console.log(sum(10)(20)(100)(500)())

 or

function sum(...args){
    let total = args.reduce((acc,val)=>acc+val,0);
    
   return function inner(...nextArgs){
        if(nextArgs.length===0){
            return total
        };
        
        total+=nextArgs.reduce((acc,val)=>acc+val,0);
        return inner;
    }
}
console.log(sum(10,10)(10)(10,10)());

32.Polyfills

Array.prototype.myFilter = function (callback){

    for(let i=0;i<this.length;i++){

            result.push(this[i]);
        }
    }
    return result;
}

58.Ways to create deep copy:
 1.JSON
   JSON.parse(JSON.stringify(obj));
 2.structuredClone(obj);
  let copy = structuredClone(obj);
  handles:objects,array,date,map,set.
 3.Manual Recursion
  function deepCopy(obj){
    if(obj!==null||typeof obj!=='object')return obj;

    let copy = Array.isArray(obj)?[]:{};

    for(let key in obj){
      copy[key] = deepCopy(obj[key]);
    }
    return copy;
  }
