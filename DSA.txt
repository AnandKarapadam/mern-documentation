1.Asymptotic notations.
 It is a mathematical tool used to describe the time and space complexity of an algorithms as input size grows.
 i.Big O notation(worst case) -> Maximum time an algorithm can take.
 ii.Omega notation(best case) -> Minimun time an algorithm can take.
 iii.Theta notation(avg case) -> Theta notation represents the exact growth rate of an algorithm.

2.Stack underflow , Stack Overflow.
Stack underflow -> Stack underflow occurs when you try to remove (pop) an element from an empty stack.
Stack Overflow(too many function calls) -> Stack overflow in JavaScript occurs when the call stack exceeds its limit, usually due to excessive or infinite recursion, resulting in a ‘Maximum call stack size exceeded’ error.
->wrong base case in recursion can trigger infinite function calls.

10.Binary Tree(file systems,searching and sorting).
 It is a tree datastructure that it can have atmost two child nodes.

16.Tail Recursion.
 A recursive function that last operation is the recursive call(no work after it).
 eg:function sum(n, acc = 0) {
     if (n === 0) return acc;
     return sum(n - 1, acc + n); // last operation = recursive call
    }

22.Adjacency Matrix vs Adjacency List vs Edge List.
 Adjacency Matrix -> Represent elements in 2d array. Fast edge lookup O(1), High memory useage(v^2), slower iteration.
 Adjacency List -> Represent elements in list. Memory efficient O(V+E), fast iteration, slower edge lookup O(degree).
 Edge List -> Simplest representation, memory efficient O(E), slow edge lookup
 
23.Why do we use arrays to represent heaps?

1. Complete Binary Tree Property
   - Heaps are complete binary trees (filled level by level)
   - This structure fits perfectly into an array without gaps

2. No Need for Pointers (Space Efficient)
   - Unlike trees, arrays don’t store left/right pointers
   - Saves extra memory → more space efficient

3. Index-based Relationships (Simple Math)
   - Parent index = (i - 1) / 2
   - Left child = 2i + 1
   - Right child = 2i + 2
   - Makes navigation very fast and simple

4. Cache Locality (Better Performance)
   - Arrays store elements in contiguous memory
   - CPU cache works efficiently  → faster access compared to pointer-based trees

5. Faster Operations
   - Insert and delete operations are efficient (O(log n))
   - No pointer traversal overhead

Conclusion:
Arrays are used for heaps because they are space-efficient, cache-friendly, and allow fast computation using index formulas.
